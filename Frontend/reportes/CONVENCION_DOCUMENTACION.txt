📌 NORMA ESTABLECIDA:

Todos los archivos de documentación (.md, .txt) creados a partir de ahora 
deberán guardarse en:

    Frontend/reportes/
    ├── md/     ← Documentación en Markdown (.md)
    └── txt/    ← Documentación en texto plano (.txt)

✅ DESDE AHORA:

1. Cada vez que crees un archivo .md o .txt para explicar algo
   → Se guardará automáticamente en reportes/md/ o reportes/txt/

2. Los archivos que YA ESTÁN en el root (README.md, package.json, etc.)
   → Se mantienen en el root (excepto si se especifica lo contrario)

3. Estructura LIMPIA del root:
   ✅ Archivos de configuración (package.json, tsconfig.json, etc.)
   ✅ README.md principal (documentación del proyecto)
   ✅ Carpetas de código (src/, public/, etc.)
   ❌ Documentación de desarrollo (va a reportes/)


📂 CLASIFICACIÓN:

MARKDOWN (.md) en reportes/md/:
  • Documentación técnica detallada
  • Guías paso a paso
  • Referencias de API
  • Arquitectura explicada
  • Ejemplos formateados
  
  Ejemplos:
  ✅ COMO_PROBAR_MOCKS.md
  ✅ MOCKS_SETUP.md
  ✅ ZUSTAND_AUTH_README.md
  ✅ AGENTS.md


TEXTO PLANO (.txt) en reportes/txt/:
  • Notas rápidas
  • Resúmenes ejecutivos
  • Guías de testing
  • Instrucciones paso a paso
  • Ejemplos de código (si es .ts/.js, también aquí)
  
  Ejemplos:
  ✅ COMO_PROBAR_ZUSTAND.txt
  ✅ MSW_SETUP_COMPLETE.txt
  ✅ ZUSTAND_ARQUITECTURA.txt
  ✅ ZUSTAND_EJEMPLOS_RAPIDOS.ts (código = .txt)
  ✅ ZUSTAND_SETUP_COMPLETE.txt
  ✅ TEST_MOCKS_QUICK.txt


🎯 CRITERIO DE ELECCIÓN:

¿Cuándo usar .md (Markdown)?
→ Si es documentación FORMAL y ESTRUCTURADA
→ Si tiene muchas secciones y subsecciones
→ Si necesita tablas, listas formateadas, enlaces
→ Si es para desarrolladores que necesitan referencia rápida
→ Ejemplo: "COMO_PROBAR_MOCKS.md" (guía completa)

¿Cuándo usar .txt (Texto plano)?
→ Si es un resumen RÁPIDO o NOTAS
→ Si es para LECTURA RÁPIDA
→ Si incluye diseños ASCII (diagramas)
→ Si es un listado de instrucciones simples
→ Ejemplo: "ZUSTAND_ARQUITECTURA.txt" (diagramas ASCII)


💾 FLUJO AL CREAR NUEVO ARCHIVO:

PASO 1: Creas un archivo de documentación
        ↓
PASO 2: Clasifica: ¿Es .md o .txt?
        ↓
PASO 3: Guárdalo directamente en:
        reportes/md/nombre.md  ← SI ES MARKDOWN
        reportes/txt/nombre.txt ← SI ES TEXTO
        ↓
PASO 4: ✅ LISTO - No vuelve a la raíz del Frontend
