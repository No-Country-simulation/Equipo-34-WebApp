🔧 CARACTERÍSTICAS IMPLEMENTADAS:
────────────────────────────────────────────────────────────────────────────────

✅ ESTADO CENTRALIZADO
   • Store único para toda la aplicación
   • Gestión de user, token, loading, error
   • Interfaz TypeScript completamente tipada

✅ PERSISTENCIA AUTOMÁTICA
   • localStorage integrado con Zustand persist middleware
   • Se guarda/restaura automáticamente
   • Nombre de clave: "auth-store"

✅ SINCRONIZACIÓN DE SESIÓN
   • AuthProvider valida token al cargar
   • Recupera usuario de /api/auth/me si existe token
   • Restaura sesión automáticamente

✅ ACCIONES DE AUTENTICACIÓN
   • login(email, password) - Autentica usuario
   • register(data) - Registra nuevo usuario
   • logout() - Cierra sesión
   • getCurrentUser() - Sincroniza con backend

✅ HELPERS DE VALIDACIÓN
   • getUserRole() - Obtiene rol actual
   • isUserRole(role) - Valida rol específico
   • isUserRole([roles]) - Valida múltiples roles

✅ 8+ HOOKS PERSONALIZADOS
   • useAuth() - Acceso completo
   • useCurrentUser() - Solo usuario
   • useIsAuthenticated() - Solo autenticación
   • useUserRole() - Solo rol
   • useHasRole() - Validar roles
   • useAuthActions() - Solo acciones
   • useAuthStatus() - Loading + error
   • useSyncCurrentUser() - Sincronización
   • useAuthDebug() - Debugging en dev

✅ PROTECCIÓN DE RUTAS
   • <ProtectedRoute> - Genérica
   • <AuthGuard> - Solo autenticación
   • <RoleGuard> - Con rol específico
   • useCanAccess() - Hook para lógica condicional

💬 PREGUNTAS FRECUENTES:
────────────────────────────────────────────────────────────────────────────────

P: ¿Dónde se guarda el token?
R: En localStorage bajo la clave "auth-store"

P: ¿Qué pasa si recargo la página?
R: AuthProvider recupera el token y valida con /api/auth/me

P: ¿Los datos se pierden si cierro el navegador?
R: No, localStorage persiste. Se restauran al abrir nuevamente.

P: ¿Cómo protejo una ruta por rol?
R: <RoleGuard roles="medico"><Componente/></RoleGuard>

P: ¿Puedo usar múltiples roles?
R: Sí: <RoleGuard roles={['medico', 'admin']}>

P: ¿Cómo veo los datos del usuario?
R: const user = useCurrentUser();

P: ¿Cómo valido el rol?
R: const isMedico = useHasRole('medico');
