 ⚡ PASO 1: PREPARACIÓN
═══════════════════════════════════════════════════════════════════════════

1️⃣  Instala MSW (si no lo tienes):
    npm install msw

2️⃣  Actualiza app/layout.tsx con MSWProvider:

    import MSWProvider from '@/mocks/providers/MSWProvider';

    export default function RootLayout({ children }) {
      return (
        <html>
          <body>
            <MSWProvider>
              {children}
            </MSWProvider>
          </body>
        </html>
      );
    }

3️⃣  Inicia el servidor:
    npm run dev

4️⃣  Verifica en console (F12):
    ✅ MSW iniciado correctamente

 🌐 MÉTODO 1: DESDE EL NAVEGADOR (DevTools Console)

PASO 1: Abre http://localhost:3000
PASO 2: Presiona F12 → Console
PASO 3: Copia y pega este código:

fetch('/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'paciente@clinic.com',
    password: 'password123'
  })
})
.then(r => r.json())
.then(data => {
  console.log('✅ Éxito:', data);
  localStorage.setItem('token', data.token);
})
.catch(e => console.error('❌ Error:', e));

═══════════════════════════════════════════════════════════════════════════
 ❌ TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════

PROBLEMA: Console dice "❌ Error iniciando MSW"
SOLUCIÓN:
  1. Verifica que MSWProvider está en app/layout.tsx
  2. Verifica que mockServiceWorker.js existe en public/
  3. Reinstala: npm install msw

PROBLEMA: Las requests no se interceptan
SOLUCIÓN:
  1. Reinicia servidor: Ctrl+C y npm run dev
  2. Limpia cache: Ctrl+Shift+R (hard refresh)
  3. Verifica que endpoint está en src/mocks/handlers/auth.ts
  4. Verifica que handlers.ts importa authHandlers

PROBLEMA: Token = null o indefinido
SOLUCIÓN:
  1. Verifica que login retorna status 200
  2. Verifica que response.json() tiene "token"
  3. Revisa Network tab para ver respuesta completa
