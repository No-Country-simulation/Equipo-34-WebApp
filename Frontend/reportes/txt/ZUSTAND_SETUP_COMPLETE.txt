🎯 Funcionalidades Implementadas:
────────────────────────────────────────────────────────────────────────────────

Store Zustand (auth.store.ts):
  ✓ Estado centralizado (user, token, isAuthenticated, isLoading, error)
  ✓ Persistencia automática en localStorage
  ✓ Acciones: login(), register(), logout(), getCurrentUser()
  ✓ Helpers: getUserRole(), isUserRole()

Hooks Personalizados (useAuth.ts):
  ✓ useAuth() - Todo el estado y acciones
  ✓ useCurrentUser() - Solo usuario
  ✓ useIsAuthenticated() - Solo autenticación
  ✓ useUserRole() - Solo el rol
  ✓ useHasRole() - Validar roles
  ✓ useAuthActions() - Solo acciones
  ✓ useAuthStatus() - Loading + error
  ✓ useSyncCurrentUser() - Sincronizar sesión
  ✓ useAuthDebug() - Debugging

Provider (AuthProvider.tsx):
  ✓ Sincroniza sesión al cargar la app
  ✓ Recupera usuario de localStorage
  ✓ Valida token con /api/auth/me

Protección de Rutas (AuthGuard.tsx):
  ✓ <ProtectedRoute> - Genérica con roles opcionales
  ✓ <AuthGuard> - Solo autenticación
  ✓ <RoleGuard> - Con rol específico
  ✓ useCanAccess() - Hook para lógica condicional

LoginForm (LoginForm.tsx):
  ✓ Validación de campos
  ✓ Botones de prueba rápida (3 roles)
  ✓ Manejo de errores
  ✓ Loading states
  ✓ Diseño responsive

Páginas:
  ✓ /auth/login - Login funcional
  ✓ /dashboard - Dashboard protegido con datos por rol


🔄 Flujo de Datos:
────────────────────────────────────────────────────────────────────────────────

[UI Login]
    ↓
[LoginForm.tsx] → useAuth().login(email, pwd)
    ↓
[auth.store.ts] → fetch('/api/auth/login')
    ↓
[MSW] → Intercepta y retorna { token, user }
    ↓
[auth.store.ts] → Guarda en estado + localStorage
    ↓
[React] → Re-renderiza componentes suscritos
    ↓
[Router] → Redirige a /dashboard
    ↓
[ProtectedRoute] → Valida autenticación
    ↓
[Dashboard] → Muestra datos con renderizado por rol

🧪 Cómo Probar:
────────────────────────────────────────────────────────────────────────────────

1. OPCIÓN UI (Más fácil):
   → Abre http://192.168.100.9:3000/auth/login
   → Haz clic en "Paciente", "Médico" o "Admin"
   → Verás el dashboard con tus datos

2. OPCIÓN Console (Debugging):
   → F12 → Console
   → Ejecuta:

   import { useAuthStore } from '@/shared/store/auth.store';
   const store = useAuthStore.getState();
   await store.login('paciente@clinic.com', 'password123');
   console.log(store.user);
   console.log(localStorage.getItem('auth-store'));

3. OPCIÓN Postman:
   POST http://192.168.100.9:3000/api/auth/login
   Body: { "email": "paciente@clinic.com", "password": "password123" }
